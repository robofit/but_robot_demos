\subsection{Detailed Description}
Base-\/class for filters. Filters allow full user-\/defined control over whether or not a message should print. The ROS\_\-X\_\-FILTER... macros provide the filtering functionality. 

Filters get a chance to veto the message from printing at two times: first before the message arguments are evaluated and the message is formatted, and then once the message is formatted before it is printed. It is also possible to change the message, logger and severity level at this stage (see the \doxyref{FilterParams}{p.}{structros_1_1console_1_1FilterParams} struct for more details).

When a ROS\_\-X\_\-FILTER... macro is called, here is the high-\/level view of how it uses the filter passed in: \begin{DoxyVerb}
if (<logging level is enabled> && filter->isEnabled())
{
  <format message>
  <fill out FilterParams>
  if (filter->isEnabled(params))
  {
    <print message>
  }
}
\end{DoxyVerb}
